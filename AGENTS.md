# AGENTS.md

Polyglot: Rust/WASM SQL transpiler for 33+ dialects (Rust core + WASM/TS SDK + Python + Go + C FFI). Note: the repo directory is named `sap-hana-sql` but the project is `polyglot`.

Toolchain is mise-managed (`mise.toml`: rust with rustfmt+clippy, opencode). Also needs pnpm 10 / Node 22+, `wasm-pack`, and `uv` (Python).

## Workspaces

- Rust workspace (`Cargo.toml`): `crates/polyglot-sql` (core), `polyglot-sql-function-catalogs`, `polyglot-sql-wasm`, `polyglot-sql-ffi`, `polyglot-sql-python`. `examples/rust` is **excluded** from the workspace.
- pnpm workspace (`pnpm-workspace.yaml`): `packages/*` (sdk, go, documentation, playground, python-docs). There is **no root `package.json`** and no root npm scripts.
- Go module lives at `packages/go` (separate `packages/go/v*` tag on release).
- Single shared version in root `Cargo.toml` `[workspace.package]` (also mirrored in pnpm packages and `packages/go/types.go`). Bump everywhere with `make bump-version V=x.y.z`.

## Test fixtures — read this before running fixture tests

SQLGlot and ClickHouse fixture tests load JSON from `crates/polyglot-sql/tests/sqlglot_fixtures/` and `tests/custom_fixtures/clickhouse/`. Both dirs are **gitignored** and must be generated first:

```bash
make extract-all-fixtures   # clones sqlglot v30.12.0 + ClickHouse into external-projects/ (gitignored), then extracts
```

Without fixtures, those tests **skip silently** (they print a warning and return) — easy to mistake for passing. Note: test source and skip messages reference `make setup-fixtures`, but **that target does not exist**; use `make extract-all-fixtures` (or `make extract-fixtures` for sqlglot only).

## Tests

```bash
make test-rust-verify          # full verification: lib + all fixtures + custom + clickhouse + FFI
make test-rust-lib             # lib unit tests only (cargo test --lib -p polyglot-sql)
make test-rust-identity        # individual fixture suites: -dialect / -transpile / -transpile-generic / -parser / -pretty
make test-rust-feature-gates   # cargo check matrix across optional feature combos
make test-ffi                  # FFI crate integration tests
make dev                       # quick: test-rust-check + test-rust
make validate                  # pre-commit: test-rust + test-compare (latter needs WASM build + Python sqlglot)
```

Single Rust test by name: `cargo test -p polyglot-sql <name> -- --nocapture`.

SDK: `cd packages/sdk && pnpm test` (vitest), `pnpm typecheck` (tsc), `pnpm lint` (biome), `pnpm run test:bundlers`.
Python: `make test-python`, `make typecheck-python` (pyright). Go: `make test-go` (integration tests skip unless `POLYGLOT_SQL_FFI_PATH` is set; use `make test-go-integration` to build FFI and run them).

### Stack/release requirements

`sqlglot_pretty`, `custom_clickhouse_parser`, and `custom_clickhouse_coverage` **must run with `--release` and `RUST_MIN_STACK=16777216`** (CI sets this; the Makefile handles it). In debug builds deeply nested SQL can stack-overflow. `.cargo/config.toml` sets `RUST_MIN_STACK=8388608` globally and a 64MB wasm stack via linker flag.

`tests/common/known_failures.rs` tracks known fixture failures and reports new passes/failures — don't delete entries without verifying.

## Generated artifacts (all gitignored — regenerate, don't edit)

- **TypeScript bindings** (ts-rs, `bindings` feature is **off by default**):
  `make generate-bindings` → runs `cargo test -p polyglot-sql --lib --features bindings export_typescript_types` → `crates/polyglot-sql/bindings/` → copied to `packages/sdk/src/generated/`.
- **WASM**: `cd packages/sdk && pnpm run build:wasm` (bundler + web targets; needs `wasm-pack`) → `packages/sdk/wasm/` + `wasm-web/`.
- **FFI header**: `crates/polyglot-sql-ffi/polyglot_sql.h`, generated by `build.rs`/cbindgen on build.

## Build

Build order matters (`make build-all`): core release → FFI → Python wheels → generate-bindings → WASM/SDK. Building the SDK before WASM + bindings are present will fail.

FFI uses a **custom profile, not `--release`**:
```bash
cargo build -p polyglot-sql-ffi --profile ffi_release   # panic=unwind; output in target/ffi_release/
```

Python uses uv + maturin from `crates/polyglot-sql-python`: `make develop-python` / `make build-python`.

`stacker` is default-on for native `polyglot-sql` (inherited by FFI/Python). The WASM crate depends on core with `default-features = false` (no `stacker`) and relies on the large wasm stack instead — do **not** add `stacker` to WASM builds.

## Format / lint

```bash
make fmt                # cargo fmt --all + cd packages/sdk && npm run format (biome)
cargo clippy --all      # clippy component pinned via mise
cd packages/sdk && pnpm lint    # biome check src/
```

## Release

Tag-driven: push a `v*` tag. CI builds + publishes to crates.io, npm, PyPI, attaches FFI archives to the GitHub Release, and deploys docs/playground/python-docs to Cloudflare Pages. The Go module gets a separate `packages/go/v*` tag. CI verifies the workspace version equals the tag, so always bump via `make bump-version V=x.y.z` before tagging.
